# 🔍 JSON Schema Contract Validator

A high-performance, batch-capable JSON Schema Validator designed for Quality Assurance (QA) and Software Development workflows. This tool allows validating **hundreds of JSON payloads** in milliseconds against standard Draft-7 schemas. 

It accepts inputs from **files, directories, JSON Lines (jsonl) files, or raw chat text**, and can be run either as a **standalone Command Line Interface (CLI)** or integrated directly into **Langflow as a Custom Component**.

## 🚀 Standalone Command Line Interface (CLI)

The CLI tool (`cli.py`) is designed for engineers and CI/CD pipelines to validate local data batches.

### CLI Options

| Argument | Short | Description |
| :--- | :--- | :--- |
| `--schema` | `-s` | Path to JSON schema file or raw schema string (Required) |
| `--input` | `-i` | Path to JSON file, directory of JSONs, `.jsonl` file, or raw JSON string (Required) |
| `--format` | `-f` | Output format: `table` (default), `json`, or `markdown` |
| `--output` | `-o` | Optional file path to save the generated report |
| `--workers`| `-w` | Number of worker threads for parallel validation (default: 4) |

### CLI Usage Examples

#### 1. Validate a Folder of JSON Files
```bash
python cli.py --schema test_data/user_schema.json --input test_data/payloads/
```

#### 2. Validate a Single JSON Lines (`.jsonl`) File (e.g. streaming or database dumps)
```bash
python cli.py --schema test_data/user_schema.json --input test_data/all_payloads.jsonl --format table
```

#### 3. Save a Markdown Report to a File
```bash
python cli.py --schema test_data/user_schema.json --input test_data/all_payloads.jsonl --format markdown --output test_data/report.md
```

#### 4. Output Raw JSON results (ideal for shell piping)
```bash
python cli.py --schema test_data/user_schema.json --input test_data/all_payloads.jsonl --format json
```

*Note: The CLI returns exit code `0` if all payloads are valid, and `1` if any payload fails validation or fails to load, making it perfect for CI/CD checks.*

## 🤖 Langflow Custom Component Integration

To use the validator inside a Langflow agent pipeline, import/register `langflow_component.py` as a custom component.

### Inputs
1. **JSON Schema** (`schema_input`): 
   - Accepts a local file path to the schema (e.g., `/path/to/schema.json`) OR 
   - Direct raw JSON schema string pasted into the multi-line input box.
2. **JSON Payload(s)** (`payload_input`): 
   - Paste a raw JSON object (e.g., `{"id": 1, "name": "John"}`) or array.
   - Paste multi-line JSON lines (jsonl).
   - Provide a file path to a single `.json` or `.jsonl` file.
   - Provide a directory path containing multiple `.json` files.
3. **Parallel Workers** (`workers`):
   - Number of thread workers used to run concurrent validation.

### Outputs
1. **Markdown Report** (`report`): 
   - A fully formatted Markdown table showing each payload's status and detailed JSON Pointer error paths. 
   - Perfect for passing directly to a Chat Output or Prompt template.
2. **JSON Results** (`results_json`): 
   - A structured dictionary containing full test summaries and error arrays. 
   - Perfect for downstream API requests or conditional logic blocks.
3. **Is Valid** (`is_valid`): 
   - Boolean output (`True`/`False`) indicating whether all payloads passed contract validation. 
   - Ideal for Routing / Conditional gates in Langflow.

### Langflow Component Code Integration
Simply copy the code from [langflow_component.py](file:///d:/ai_3x_qa/QA_AI_Projects/AI_tools/langflow-agents/contract-validator/langflow_component.py) and paste it into a **Custom Component** node in Langflow. It includes path fallback checks to automatically locate `validator.py` on the workspace path.

## 🏎️ Verification and Performance Testing

To verify the validator's performance on your local system, run:

```bash
python verify.py
```

### Performance Metrics (Typical Run)
- **Individual Files Load & Validation**: 150 files loaded and validated in **~0.03 seconds** (using 4 parallel threads).
- **JSON Lines File Load & Validation**: 150 payloads loaded and validated in **~0.015 seconds**.
- **Error Granularity**: Captures missing fields, type constraints (e.g. integer vs string), value ranges (minimum/maximum limits), format constraints (regexes/emails), and collection criteria (uniqueItems) indicating exact JSON Pointers (e.g. `/tags/0` or `/age`).
